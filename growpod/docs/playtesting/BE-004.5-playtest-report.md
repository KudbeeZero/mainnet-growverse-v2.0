## 3. Passes

All core-loop, economy, feature-gating, and error-handling cases above passed. Highlights
with observed values from the live run:

- **Full loop** — seed → flowering (cared-for, alive) → harvest **216.5 g** → cure raised
  quality **13.53 → 23.53** → sell **602.94 GROW**, balance **145.0 → 747.94** (delta ==
  sale value exactly).
- **No-double-faucet** — second sell of the same harvest → `400`.
- **Replant** — full second cycle completed (harvest+sell `201`).
- **Feature-gating (fail-closed)** — flags default ON; setting `FEATURE_MARKETPLACE=false` /
  `FEATURE_UNIVERSITY=false` makes their routes return `404` while `grow_chamber` stays ON.
- **Economy health** — `reconciled: true`, `money_supply == ledger_net` (e.g. `1111.04`).
- **Auth** — unauth write `401`, bad key `403`, missing resource `404`.

## 4. Failures

**No game/backend defects found.** Two driver checks reported non-pass and were
investigated to ground truth:

| Driver check | Reported | Ground truth | Verdict |
|---|---|---|---|
| `daily stipend claim 200` | got `201` | `201 Created` is the correct success for a claim that creates a ledger grant; the second same-day claim correctly returns `400` | **Driver assertion too strict** — not a bug |
| `flags default ON` | values not all `True` | The endpoint nests the map as `{"flags": {...}}`; the inner map *is* all-`True`. The driver inspected the outer wrapper | **Driver assertion bug** — not a bug |

Both are documented here for transparency; neither indicates a product issue.

## 5. Reproduction Steps

```bash
# from repo root, venv with deps (make setup)
GROW_TEST_CLOCK=true APP_ENV=development \
  .venv/bin/python -c "see docs/playtesting/BE-004.5-playtest-report.md driver appendix"

# full regression
make test            # 283 passed, 84.63% coverage
```

Live driver (used for this report) seeds a throwaway SQLite DB, enables the dev clock, then
drives over the Flask test client:

1. `POST /api/game/players` → capture `id` + `api_key`.
2. `GET .../wallet` (with `X-API-Key`) → starter balance.
3. `POST .../seeds/buy`, `POST .../pods`, `POST .../plant`.
4. `POST .../pods/<pod>/environment`; loop `water` + `feed` + `POST /api/dev/clock/advance {days:3}` until `growth_stage ∈ {flowering, harvest}`.
5. `POST .../harvest {sell:false}` → `POST .../cure {target_hours:72}` → `cure/finish` (400 before, advance clock 80h, 200 after) → `POST .../sell`.
6. `GET /api/game/flags`, `GET /api/game/economy/health`; error probes (no key, bad key, bad id, double actions).

Gating repro: set `FEATURE_MARKETPLACE=false FEATURE_UNIVERSITY=false` before `create_app`;
`GET /api/game/market` and `/api/game/university/catalog` → `404`.
