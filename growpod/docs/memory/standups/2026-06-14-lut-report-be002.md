## 0) One-paragraph summary
Built a **dev/test-only simulation test clock** to unblock grow-loop testing and launch readiness,
without touching the economy or real players. Because the engine is already **compute-on-read**
(state is a pure function of stored state + `clock.now()`), the clock is just an `OffsetClock` —
wall time plus a mutable, **forward-only** offset — layered on the existing `Clock` seam. Advancing
it makes the next read of any plant catch up through the normal `engine.catch_up`, so there is **zero
new simulation logic**. A dev-only blueprint (`/api/dev/clock/{,advance,reset}`) is registered **only**
when `GROW_TEST_CLOCK=true` on a non-production `APP_ENV`, and force-disabled in production.

## 1) What shipped
- **`simulation/clock.py`** — `OffsetClock` (forward-only `advance`, `reset`, `offset`), process
  singleton `get_test_clock`/`reset_test_clock`, and `active_clock()` selector (shared test clock
  when enabled, else `SystemClock`).
- **`config.py`** — new `APP_ENV` setting + `is_production`; `test_clock_enabled = GROW_TEST_CLOCK
  AND not production` (force-disabled in prod).
- **`services/simulation_service.py`** — default clock now resolves through `active_clock()` (explicit
  injection still wins); no per-call-site changes needed for reads to pick up the shift.
- **`api/dev_api.py`** (new) — `GET /api/dev/clock`, `POST /clock/advance` (hours/days, >0, ≤8760h,
  syncs living plants), `POST /clock/reset`. Per-request re-guard → 404 when disabled.
- **`api/flask_api.py`** — registers the dev blueprint only when `test_clock_enabled`.
- **Docs** — ADR (`DECISIONS.md` 2026-06-14), usage (`docs/SIMULATION_TEST_CLOCK.md`), BACKLOG
  Launch-Readiness path (STEP 3 ✅, STEP 4 ⬜).
- **Tests** — `tests/test_test_clock.py` (15): OffsetClock primitive, config gating
  (off by default / on in dev / force-off in prod), `active_clock()` selector, endpoints
  (absent when disabled, advance progresses a plant, **no economy mutation**, reset).
